# 測試說明文件

本文檔說明如何使用合併後的測試腳本 `test_all_tests.py`。

## 快速開始

### 執行所有測試
```bash
python test_all_tests.py
```

### 執行特定測試
```bash
python test_all_tests.py --test director     # 僅執行總監整合測試
python test_all_tests.py --test pipeline      # 僅執行動態篇卷管線測試
python test_all_tests.py --test feedback      # 僅執行反饋環路測試
python test_all_tests.py --test integrity     # 僅執行完整性校驗測試
python test_all_tests.py --test rebound       # 僅執行管線反彈測試
python test_all_tests.py --test retrospective # 僅執行回溯日誌測試
python test_all_tests.py --test simple        # 僅執行簡單日誌測試
```

### 列出所有測試
```bash
python test_all_tests.py --list
```

### 跳過特定測試
```bash
python test_all_tests.py --skip-director --skip-simple
```

## 測試項目說明

| 測試名稱 | 鍵值 | 說明 |
|---------|------|------|
| 總監整合測試 | `director` | 測試總監預檢查 (`pre_check_next_agent`) 與提示生成 (`get_simplified_director_prompt`) 功能。需要真實資料庫 `novel_factory.db`。 |
| 動態篇卷管線測試 | `pipeline` | 測試動態篇卷結構、章節範圍計算、增量局部修改功能。包含 5 個子測試項目。 |
| 反饋環路測試 | `feedback` | 測試反饋環路機制、世界觀修補 (`add_worldview_patch`)、下游標記 (`mark_downstream_dirty`)、JIT 對齊功能。 |
| 完整性校驗測試 | `integrity` | 測試總監大綱與情節邏輯校驗 (`verify_novel_integrity`)，包含：<br>1. 完美情境大綱對齊測試<br>2. 缺陷情境（伏筆憑空回收、時序顛倒、未回收等）警告檢驗<br>3. **新增：早期設定階段（如 `init` 階段）無死結校驗（大綱為空回傳「不適用」報告而不報警阻斷流程）**。 |
| 管線反彈測試 | `rebound` | 測試創作管線對齊、世界觀結構持久化、多態角色解析、進度比例映射演算法。 |
| 回溯日誌測試 | `retrospective` | 測試回溯 API (`api_novel_retrospective`) 的終端機日誌輸出功能，使用 Mock 避免實際 API 呼叫。 |
| 簡單日誌測試 | `simple` | 手動呼叫 retrospective 並觀察終端機輸出。這是整合測試，需要較長時間。 |

## 測試配置

測試腳本支援以下超參數配置：

```python
TEST_CONFIGS = {
    "director": {
        "name": "總監整合測試 (Director Integrity)",
        "description": "測試總監預檢查與提示生成功能",
        "db_path": "novel_factory.db"  # 可自訂資料庫路徑
    },
    "pipeline": {
        "name": "動態篇卷管線測試",
        "description": "測試動態篇卷與對齊功能",
        "requires_mocks": False
    },
    # ... 其他配置
}
```

## 環境需求

- Python 3.8+
- SQLite3
- 專案模組：`db.py`, `agents.py`, `agents_incremental.py`, `app.py`, `llm.py`

## 注意事項

1. **總監整合測試** 需要已存在的 `novel_factory.db` 資料庫，若不存在會自動跳過。
2. **簡單日誌測試** 會實際呼叫 API，可能需要較長時間，預設會執行。
3. 所有測試結束後會自動清理測試資料庫記錄。
4. 使用 `--skip-xxx` 參數可以選擇性跳過不需要的測試。

## 疑難排解

### 編碼問題
若在 Windows 上遇到 CP950 編碼錯誤，腳本已自動設定 UTF-8 輸出：
```python
if sys.stdout.encoding != 'utf-8':
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
```

### 模組匯入錯誤
確保在專案根目錄執行腳本，或確認 `sys.path` 包含正確路徑。

### 資料庫連線失敗
檢查 `novel_factory.db` 是否存在於目前目錄，或修改 `TEST_CONFIGS["director"]["db_path"]` 指向正確路徑。
