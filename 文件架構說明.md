# AI Novel Factory 系統架構說明文檔

## 📋 文檔概述

本系統是一個 AI 小說創作工廠，採用多智能體（Multi-Agent）架構，透過前端的 JavaScript 管道系統與後端的 Python Agent 模組協同工作，實現從世界觀設計到正文寫作的完整創作流程。

---

## 🧠 Python 和 JavaScript 的分工

> **JavaScript = 導演（控制流程和UI）**
> **Python = 演員（實際執行AI創作）**

| 職責 | JavaScript (前端) | Python (後端) |
|------|------------------|---------------|
| **主要職責** | 控制「什麼時候做什麼」 | 執行「實際的創作工作」 |
| **UI 界面** | ✅ 管理所有介面更新 | ❌ 不碰 UI |
| **流程控制** | ✅ 決定執行順序 | ❌ 不知道下一步去哪 |
| **AI 實際生成** | ❌ 只負責呼叫 | ✅ 實際呼叫 LLM |
| **資料儲存** | ❌ 觸發儲存 | ✅ 執行 SQLite 寫入 |
| **即時回饋** | ✅ SSE 接收並顯示 | ❌ 只負責發送 SSE |

## 📁 目前活躍的 4 個 Agent 文件

```
agents.py              → Facade 統一入口
agent_main.py          → 主要創作邏輯
agent_assembler.py     → Prompt 參數組裝器
agent_extension.py     → 總監決策 + 增量修改
```

### 各文件職責

| 文件 | 行數 | 職責 |
|------|------|------|
| `agents.py` | ~70 | Facade 包裝器，統一對外暴露所有函數 |
| `agent_main.py` | ~1400+ | 主要創作邏輯（worldview, characters, volumes, plot, writer） |
| `agent_assembler.py` | ~1000+ | Prompt 組裝器，統一呼叫 LLM |
| `agent_extension.py` | ~700 | 總監決策 + 增量修改 + JIT 對齊 |

## 📝 前端 Pipeline 系統（JavaScript）

### static/pipeline.js
- **核心功能**：協調所有創作階段的執行順序與進度追蹤
- **創作流程**：`worldview → characters → volumes → volume_skeleton → foreshadowing_orchestration → plot → writer`

### static/pipeline_logic.js
- **設計理念**：函數式程式設計，無副作用的純函數
- **核心函數**：`getPipelinePrompt()`, `resolveNextStageFromDecision()`

### static/pipeline_logic.test.js
- **測試覆蓋**：`getPipelinePrompt` (5個) + `resolveNextStageFromDecision` (14個)

## 🎯 總監決策系統

### verify_novel_integrity() 完整性校驗維度

| 維度 | 檢查項目 |
|-----|---------|
| 📂 設定結構 | multi_act_structure、progressive_character_plan |
| 📊 數量檢驗 | 卷數、章節序號連續性、佔位大綱 |
| 🌍 世界觀使用率 | 陣營/勢力、伏筆種子、轉折點使用率 |
| 🔑 伏筆時序 | Dangling Plants、Baseless Payoffs、Out of Order |
| ✍️ 寫作連貫性 | 與前文、大綱、世界觀的一致性 |
