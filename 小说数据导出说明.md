# 小说数据导出功能说明

## 概述

本说明文件解释 AI Novel Factory 系统中如何将小说数据完整导出为文本文件。系统已具备完整的数据导出能力，但前端 UI 尚未提供导出按钮。

---

## 📊 数据存储架构

### 后端数据库结构
小说数据分散存储在以下表中：
- `novels` - 小说基本信息（标题、类型、风格）
- `worldbuilding` - 世界观设定（版本化管理）
- `characters` - 角色圣经（JSON 格式，版本化管理）
- `plot_chapters` - 剧情章节大纲（JSON 格式）
- `chapters` - 正文章节内容（每章独立存储）
- `volumes` - 篇卷规划信息

### 数据获取方式
通过 `GET /api/novels/{novel_id}` 获取小说完整数据：
```json
{
  "novel": { ... },
  "worldbuilding": "世界观内容",
  "worldbuilding_version": 1,
  "characters": { "parsed_data": {...} },
  "characters_raw": "原始JSON",
  "characters_version": 1,
  "plot": { "chapters": [...] },
  "plot_raw": "JSON 字符串",
  "chapters": [ ... ],  // 所有已写章节
  "volumes": [ ... ],
  "worldview_patches": [ ... ]
}
```

## 🚀 现有导出 API

### 端点
```
GET /api/novels/{novel_id}/export?format=txt|markdown
```

### 支持的格式

#### 1. TXT 格式 (`format=txt`)
- **用途**：纯文本格式，适合阅读和分享
- **内容结构**：
  ```
  《小说标题》
  题材：类型
  風格基調：风格
  =========================================
  
  【第 X 章：章节标题】
  
  章节正文内容
  
  ------------------------------------------
  ```
- **编码**：UTF-8
- **文件名**：`{标题}_完整正文.txt`

#### 2. Markdown 格式 (`format=markdown`)
- **用途**：结构化文档，适合转换为其他格式（PDF、EPUB 等）
- **内容结构**：
  ```markdown
  # 《小说标题》
  
  - **题材**: 类型
  - **風格基調**: 风格
  
  ---
  
  ## 📖 世界觀與核心設定
  
  {世界構内容}
  
  ---
  
  ## 👥 角色聖經 (Character Bible)
  
  {JSON 格式角色数据}
  
  ---
  
  ## 🗺️ 劇情章節大綱
  
  {JSON 格式章节大纲}
  
  ---
  
  ## 📝 小說完整正文
  
  ### 第 X 章：章节标题
  
  章节正文内容
  ```
- **编码**：UTF-8
- **文件名**：`{标题}_小說設定與全書正文.md`

## 🖥️ 前端调用方法

### 直接浏览器访问
在浏览器地址栏中输入：
```
http://localhost:8000/api/novels/{novel_id}/export?format=txt
```
或
```
http://localhost:8000/api/novels/{novel_id}/export?format=markdown
```
浏览器会自动触发下载。

### JavaScript 调用示例
```javascript
// 在页面中动态创建下载链接
function downloadNovel(novelId, format = 'txt') {
  const token = localStorage.getItem('auth_token'); // 如果有认证
  const url = `/api/novels/${novelId}/export?format=${format}`;
  
  // 方法 1: 直接打开（简单）
  window.open(url, '_blank');
  
  // 方法 2: 创建隐藏链接并点击
  const link = document.createElement('a');
  link.href = url;
  link.download = ''; // 让服务器决定文件名
  document.body.appendChild(link);
  link.click();
  document.body.removeChild(link);
}

// 使用示例
downloadNovel('your-novel-id-here', 'markdown');
```

## 🔧 后端处理逻辑

### 数据收集流程（app.py 第 688-752 行）

1. **获取小说基本信息**
   ```python
   novel = db.get_novel(novel_id)
   wb = db.get_latest_worldbuilding(novel_id)
   char = db.get_latest_characters(novel_id)
   plot = db.get_latest_plot_chapters(novel_id)
   chapters = db.get_all_chapters_latest(novel_id)
   ```

2. **TXT 格式生成**
   - 拼接标题、题材、风格
   - 按章节索引排序所有章节
   - 每章输出：章节标题 + 正文内容 + 分隔线
   - 返回纯文本流

3. **Markdown 格式生成**
   - 使用 Markdown 标题和强调语法
   - 包含：小说元信息、世界观、角色JSON、剧情大纲、正文
   - 每章使用 `###` 三级标题
   - 返回 Markdown 文本流

4. **响应头设置**
   ```python
   headers = {
       "Content-Disposition": f"attachment; filename*=utf-8''{quote(filename)}"
   }
   return Response(
       content=content,
       media_type="text/plain; charset=utf-8"  # 或 "text/markdown"
   )
   ```

## 📝 注意事项

1. **认证需求**：当前系统未启用认证，导出 API 可直接访问。如需保护数据，建议添加登录验证。
2. ** Novel ID 获取**：前端通过 `state.currentNovelId` 获取当前小说标识。
3. **大文件处理**：如果小说章节非常多，导出文件可能较大，建议显示进度提示。
4. **编码问题**：后端已正确处理 UTF-8，前端无需额外编码转换。
5. **章节排序**：导出时按 `chapter_index` 升序排列，确保章节顺序正确。
